# Brew GIS Column Naming Convention

## Principles

1. **Terse over verbose.** Short names are easier to type, read, and extend in
   SQL, macros, and cross-references: `pop` not `population`, `hh` not
   `households`, `du` not `dwelling_units`, `emp` not `employment`.
2. **Consistent prefixing.** Sub-type columns always use the parent name as a
   prefix: `du_detsf`, `du_mf`, `emp_ret`, `emp_off`.
3. **Snake_case only.** PostgreSQL unquoted identifiers are case-folded to
   lower case; we match that.
4. **Unit in name when ambiguous.** `_sqft`, `_acres`, `_kwh`, `_kg`, `_af`
   suffixed where raw count would be ambiguous.
5. **No Hungarian notation.** No `f_`, `i_`, `str_` prefixes.

## Key Identifiers

These four columns form the identity/addressability layer of every parcel
record. They are **not interchangeable.**

| Canonical Name | Aliases (current codebase) | Definition |
|---|---|---|
| `id` | `parcel_id` | **SERIAL PK.** Autogenerated primary key on the base canvas table. dbt models universally alias this as `parcel_id`. The seed file `test_parcels.csv` also calls the first column `parcel_id`. |
| `geography_id` | — | **Integer.** Foreign key to census geography (block group FIPS code as int). Used for joining ACS, LEHD, NLCD source data. May be NULL for synthetic or non-census parcels. |
| `geometry_key` | — | **VARCHAR(128).** External parcel identifier from source data (e.g. APN, assessor parcel number). Currently unused in seeds (always NULL); reserved for upstream parcel tracking. |
| `id_source` | — | **VARCHAR(64).** Source system that produced the record. Currently NULL in test data. |

### Mapping rules

| Context | Preferred Name | Notes |
|---|---|---|
| Base canvas table (DDL) | `id` (SERIAL PK) | Auto-increment primary key |
| dbt models (SELECT output) | `parcel_id` | Every dbt model uses this name for the PK column |
| dbt `sources.yml` | `id` | The parcel table's key column in raw source |
| Canvas views | `parcel_id` | Dynamic SQL views use this for join consistency |
| Spatial allocator | matches source | Uses whatever name the source table provides via SQLAlchemy |
